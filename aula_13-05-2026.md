# Relatorio de produtos em PDF

Objetivo da aula: criar uma feature simples de relatorio com filtros, usando `DB::table()` para consultar os produtos e gerar download em PDF.

## 1. Instalar o pacote de PDF

```bash
composer require barryvdh/laravel-dompdf
```

Esse pacote permite gerar PDF a partir de uma view Blade.

## 2. Criar as rotas

Arquivo: `routes/web.php`

```php
Route::get('/products/report', [ProductsController::class, 'report'])->name('products.report');
Route::get('/products/report/pdf', [ProductsController::class, 'reportPdf'])->name('products.report.pdf');
```

A primeira rota mostra a tela de filtro. A segunda gera e baixa o PDF.

## 3. Importar as classes no controller

Arquivo: `app/Http/Controllers/ProductsController.php`

```php
use Barryvdh\DomPDF\Facade\Pdf;
use Illuminate\Support\Facades\DB;
```

`Pdf` gera o arquivo PDF. `DB` permite consultar o banco com Query Builder.

## 4. Criar a tela de filtro

Metodo:

```php
public function report()
{
    return view('products.report', [
        'types' => Type::orderBy('name')->get()
    ]);
}
```

Essa tela recebe os tipos para montar o campo `select`.

## 5. Criar a consulta com DB::

Metodo:

```php
public function reportPdf(Request $request)
{
    $products = DB::table('products')
        ->join('types', 'products.type_id', '=', 'types.id')
        ->select(
            'products.id',
            'products.name',
            'products.description',
            'products.quantity',
            'products.price',
            'products.image',
            'products.type_id',
            'types.name as type_name',
            'products.created_at',
            'products.updated_at'
        )
        ->when($request->name, fn ($query, $name) => $query->where('products.name', 'like', "%{$name}%"))
        ->when($request->type_id, fn ($query, $typeId) => $query->where('products.type_id', $typeId))
        ->when($request->min_quantity, fn ($query, $quantity) => $query->where('products.quantity', '>=', $quantity))
        ->when($request->max_quantity, fn ($query, $quantity) => $query->where('products.quantity', '<=', $quantity))
        ->orderBy('products.name')
        ->get();

    return Pdf::loadView('products.report-pdf', compact('products'))
        ->download('relatorio-produtos.pdf');
}
```

Pontos importantes:

- `DB::table('products')` inicia a consulta na tabela de produtos.
- `join('types', ...)` busca o nome do tipo.
- `select(...)` define os campos do relatorio.
- `when(...)` aplica filtros somente quando o campo foi preenchido.
- `orderBy('products.name')` ordena por nome em ordem crescente.
- `Pdf::loadView(...)` transforma uma view Blade em PDF.

## 6. Criar a view do filtro

Arquivo: `resources/views/products/report.blade.php`

Essa view usa componentes do Breeze:

```blade
<x-input-label />
<x-text-input />
<x-primary-button />
```

Campos usados:

- nome;
- tipo;
- quantidade minima;
- quantidade maxima.

O formulario usa `method="GET"` para enviar os filtros pela URL.

## 7. Criar a view do PDF

Arquivo: `resources/views/products/report-pdf.blade.php`

Essa view e um HTML simples com tabela. Ela lista:

- id;
- nome;
- descricao;
- quantidade;
- preco;
- imagem;
- type_id;
- nome do tipo;
- created_at;
- updated_at.

## 8. Adicionar o botao na listagem

Arquivo: `resources/views/products/index.blade.php`

Foi adicionado um link acima da tabela:

```blade
<a href="{{ route('products.report') }}">Relatorio</a>
```

Esse botao abre a tela de filtros.

## Fluxo final

1. Usuario acessa `Produtos`.
2. Clica em `Relatorio`.
3. Preenche os filtros.
4. Clica em `Exportar PDF`.
5. O Laravel consulta com `DB::table()`.
6. O PDF e baixado como `relatorio-produtos.pdf`.
